## Appendix
### A. プロンプトテンプレート / Prompt Templates
1. **UI自動化ワークフロー**
   - モック受領 → 実装指示 → スナップショット生成 → 視覚比較 → レポート提示。
   - 例: `このデザインを実装してください。`（モック画像添付） → 実装完了後に`make generate-snapshots`を実行し、視覚差分を確認する。
   - 例: `この画面のUIを実装してください。実装後、スナップショットを生成して視覚的に検証してください。`
   - 検証結果はTesting & Validationに「Snapshot/Visual」で記録し、必要なら画像パスをリンク。
2. **大規模リファクタリング**
   - `agents.md`ルール確認 → 影響範囲列挙 → テスト戦略 → フェーズ提案 → `plans.md`更新。
   - プロンプト例: `この機能の実装計画を plans.md に書いてください。添付ドキュメントを参照し、詳細なExec Planを作成してください。`
   - 生成した計画を基に、進捗ログ・決定ログ・発見事項を逐次更新し、必要に応じて`watch -n 1 "cargo test"`や`npm test -- --watch`で常時テストを回す。
3. **コードレビュー**
   - 差分要約 → リスク推定 → `claudecode /review`コマンド実行 → フィードバック反映ログ更新。
   - レビュー修正ループ: `/review` → 指摘修正 → `/review`再実行 → 問題解消 → `git add . && git commit ... && git push`。
   - GitHub連携が有効な場合は自動PRレビューが実行されるため、結果をReview Logに反映する。

### B. 用語集 / Glossary
- バイブコーディング (Vibe Coding): Codexと人間がテンポよく意思疎通しながら開発を加速するスタイル。
- 最後の10%問題: 実装完了直前に残る細かな仕上げタスク群。Codexが明示的に追跡し、必要に応じて委任する。
- Exec Plan: Codexが作業を進めるための計画台帳。`plans.md`で管理する。
