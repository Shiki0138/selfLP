OpenClaw活用版。1人運営 + AI自動化を前提とした実行計画。2026年3月策定。

## v1からの変更点

v1は「1人で週9時間」を上限に施策を絞り込んだ。v2ではOpenClawを導入することで、手動でやっていた反復作業を自動化し、同じ週9時間でカバーできる施策の範囲を広げる。

変わるのは「やること」ではなく「やり方」。戦略の骨格（サロン卸 + Amazon + 公式EC + SNS/広告）は変えず、実行速度と精度を上げる。

## OpenClawとは何か

ローカルで動く自律型AIエージェント。LINE、WhatsApp、Telegram、Slackなどのメッセージアプリに直接接続し、Web閲覧、シェルコマンド実行、ファイル操作、マルチステップのワークフロー自動実行が可能。コミュニティに5,400以上のスキルが登録されている。

本体は無料（OSS）。コストはLLM APIの利用料で、軽い使い方なら月1.5-3万円、ヘビーに使っても月10-15万円程度。

デプロイはCloudflare Workers経由が最も簡単で、インフラ費用は月1,500円以下。

selfにとって重要なのは、LINEにネイティブ対応している点。サロン向けLINE、消費者向けLINEの両方で、メッセージ受信をトリガーにした自動応答・ワークフローが組める。

## OpenClawで自動化する5つのワークフロー

### ワークフロー1: 競合広告リサーチ（施策5）

CPA 8,130円の最大の原因は、クリエイティブの訴求軸が市場とずれている可能性が高い。競合が何をやっているかを知らないまま広告を回しても改善しない。

OpenClawにMeta Ad Libraryを定期巡回させ、ヘアケア競合ブランド（YOLU、&honey、ボタニスト、MEDULLA、BULK HOMME等）の現在アクティブな広告を収集する。抽出するのはヘッドライン、本文コピー、CTA、クリエイティブの形式（静止画/動画/カルーセル）、訴求軸（成分/価格/Before After/口コミ）。

週1回の自動実行で、Googleスプレッドシートに追記していく。自分はそのシートを月曜に5分眺めるだけ。市場で効いている訴求軸が見えれば、自社クリエイティブに反映できる。

設定工数: 初回2時間
運用工数: 週5分（シート確認）
期待効果: Meta広告のCPA改善速度が2-3倍に

### ワークフロー2: 広告コピーの量産（施策6）

Meta広告のクリエイティブ疲弊が起きるたびに手動で新しいコピーを考えるのは非効率。OpenClawに以下の情報を常時参照させる。

- LP（self138.com）の全コンテンツ
- 商品情報（成分、価格、実績）
- 競合リサーチの結果（ワークフロー1）
- 過去の広告パフォーマンスデータ

「ヘッドライン5案 + 本文3案 + CTA2案」を週1回自動生成。LINE（またはSlack）に送ってくるので、良いものを選んでMeta Ads Managerに入稿するだけ。

設定工数: 初回1時間
運用工数: 週15分（選定+入稿）
期待効果: クリエイティブ制作時間を週30分 → 15分に短縮。テスト速度が倍になる

### ワークフロー3: サロンLINE自動応答（施策7）

サロン向けLINE公式にOpenClawを接続し、以下を自動処理する。

- 「発注」というキーワードを含むメッセージ → 発注フォームのURLを自動返信
- 「在庫」「残り」というキーワード → 在庫確認方法を案内
- 月1回の定期配信 → トークスクリプト+販売事例を自動送信
- 未読メッセージのサマリーを毎朝LINEに通知（自分宛て）

美容師からの問い合わせに即レスできるようになる。返信が遅れて機会損失していた分を回収できる。

設定工数: 初回3時間
運用工数: 週15分（通知確認+手動対応が必要なものだけ返信）
期待効果: サロン対応時間を週30分 → 15分に短縮。再発注率の向上

### ワークフロー4: LINE髪質診断の自動化（施策13）

消費者向けLINEにOpenClawを接続し、髪質診断を完全自動化する。既存のline-diagnostic-flow.mdの5問フロー設計をそのまま実装できる。Lステップ（月額2,980円〜）やエルメ（月額10,780円〜）が不要になる。

友だち追加 → ウェルカムメッセージ → 5問の診断（ボタン選択） → 4タイプの結果 + 商品レコメンド → ステップ配信（0日/1日/3日/5日/7日/10日/14日）

OpenClawはLINEのMessaging APIに直接接続できるため、リッチメニュー操作、セグメント分け、条件分岐、ステップ配信のすべてをコードベースで実装可能。Lステップの月額費用を節約しつつ、より柔軟なカスタマイズができる。

ただしPhase 3（5ヶ月目以降）で導入。理由は、友だち数が少ない段階で高度な自動化を組んでもリターンが小さいから。友だち200人を超えたタイミングで着手する。

設定工数: 初回5時間
運用工数: 週15分（結果確認+CVR分析）
期待効果: 診断→購入CVR 15%。Lステップ月額2,980-10,780円の節約

### ワークフロー5: UTM監査と広告レポート自動化

現在LPに実装済みのUTM引き継ぎ機能を活用し、OpenClawに以下を自動化させる。

- 全広告のUTMパラメータの整合性チェック（週1回）
- Meta Ads Manager + Amazon広告 + GA4のデータを1枚のレポートに統合（週1回）
- KPIが異常値（CPA 5,000円超、ACOS 35%超など）に達した場合、LINEに即時アラート

現状は各プラットフォームを個別にログインして確認しているが、これを1つのサマリーにまとめることで、月曜の広告確認作業が30分 → 10分に短縮。

設定工数: 初回2時間
運用工数: 週10分
期待効果: 広告運用の判断速度向上。異常値の早期検知

## OpenClaw導入のロードマップ

### Week 1: 環境構築 + ワークフロー1（競合リサーチ）

OpenClawをCloudflare Workers経由でデプロイ。LINEのMessaging APIキーを取得してLINE接続を確認。まずは最もリスクが低い「競合広告リサーチ」から稼働させる。

必要なもの:
- Cloudflareアカウント（無料）
- Claude APIキー（既にあるはず）
- LINEデベロッパーアカウント（無料）
- 作業時間: 3-4時間

### Week 2: ワークフロー2（広告コピー生成）+ ワークフロー5（レポート）

競合リサーチが動いていることを確認したら、広告コピー生成とレポート自動化を追加。この2つは読み取り+生成だけなので、外部サービスを壊すリスクがない。

### Week 3-4: ワークフロー3（サロンLINE自動応答）

サロン向けLINE公式アカウントを開設し、OpenClawを接続。最初は自動応答のルールを最小限にし、実際のメッセージを見ながら調整していく。

### Month 5以降: ワークフロー4（LINE髪質診断）

LINE友だち200人超えた段階で着手。

## 改訂版 6ヶ月ロードマップ

### Phase 1: 基盤 + AI自動化の立ち上げ（1-2ヶ月目）目標月商200-300万円

やることは5つ。前半3つは手動、後半2つはOpenClawの初期設定。

1. サロン100店舗の分類と再連絡（手動、初週3時間）

100店舗の最終発注日をスプレッドシートに整理。A群（月3万円以上/10-15店舗）、B群（不定期/20-30店舗）、C群（半年以上音沙汰なし/50-60店舗）。C群50店舗に再連絡を入れる。10店舗が再稼働すれば月10-30万円の上積み。

再発注フォームはGoogleフォームで3タップ完了。店販トークカードをCanvaで1枚作り、再発注時に同梱。

2. Amazon最適化（手動、初回2時間で完了）

タイトル・箇条書きの書き換え。既存ドキュメントの改善案をそのまま実行。否定キーワードの一括登録。SP広告3キャンペーン開始（月3-5万円）。同梱レビューカード発注（ラクスル1,000枚3,000円）。FBA移行。

3. Meta広告の再構築（手動、初回1時間）

既存キャンペーン全停止。購入最適化1キャンペーン・1広告セット・3クリエイティブ。日予算2,000-3,000円。

4. OpenClaw環境構築 + 競合リサーチ自動化（初回4時間）

Cloudflare Workersにデプロイ。Meta Ad Libraryの競合広告を週次で自動収集。YOLU、&honey、ボタニスト、MEDULLA等のアクティブ広告をスプレッドシートに蓄積。

5. 広告コピー自動生成 + レポート自動化（初回3時間）

LP情報と競合データを食わせ、週1回の広告コピーバリエーション生成。Meta + Amazon + GA4の統合レポートを毎週月曜に自動送信。KPI異常値アラート設定。

## OpenClaw試作の進め方

まず1つだけ動かして手触りを掴む。全5ワークフローを一度に作ろうとしない。

### Step 1: 競合広告リサーチ（最も安全な試作）

これを最初にやる理由は、外部サービスへの書き込みが一切ないから。Meta Ad Libraryを読むだけ。壊れるものがない。

必要な手順:
1. OpenClawをローカルまたはCloudflareにインストール
2. Claude APIキーを設定
3. 「Meta Ad Library skill」をインストール（コミュニティレジストリから）
4. 監視対象ブランド名のリストを設定
5. 出力先をGoogleスプレッドシートに設定
6. 週1回のスケジュール実行を設定

ここまでで2-3時間。動作確認できたら、翌週にワークフロー2（広告コピー生成）を追加。

### Step 2: 広告コピー生成

LP（self138.com）のHTMLをOpenClawに読み込ませ、商品情報と競合データを組み合わせて広告コピーを生成する。出力はLINEに送信。

### Step 3以降

サロンLINE自動応答 → レポート自動化 → LINE髪質診断の順で追加。

## セキュリティに関する注意

OpenClawは2026年2月に重大な脆弱性（悪意あるWebサイトによるエージェント乗っ取り）が報告され、パッチ済み（v2026.2.25）。ローカルでファイルやAPIキーにアクセスできるため、以下を厳守する。

- 常に最新版を使う
- APIキーの権限を最小限にする（LINE Messaging APIは送信のみ、読み取り専用など）
- OpenClawに渡す権限を明示的に制限する
- 本番のEC管理画面（Wix、Amazon Seller Central）への書き込み権限は渡さない
- 最初は読み取り+生成のワークフローだけ動かし、書き込み系は十分理解してから
